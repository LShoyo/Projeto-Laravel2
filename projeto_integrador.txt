Projeto Integrador 

- Os alunos deverão se separar em grupos de até 4 pessoas.

- Os grupos terão a possibilidade de criar projetos relacionados a quase todo tipo de assunto (deverão conversar com os professores antes para colocar em pauta o projeto que estão em mente), mas,
oferecemos como pontapé inicial três projetos (quizz, rede social, e-commerce).

- O projeto integrador pode ser comparado a um TCC de graduação, porém, simplificado. Onde durante estes meses de imersão ao longo do curso de desenvolvimento web full stack, os grupos terão que cumprir etapas (sprints), que irão incrementar o projeto. Para que no final do curso tenham um projeto bacana para apresentar a banca (professores: Victor e Marcelo, no caso, desta turma e coordenadora: Ana Paula)

- As sprints seguem esta ordem: Frontend, Banco de Dados, Backend. 
A primeira sprint (Frontend), os grupos terão que desenvolver as páginas HTML para mostrar aos professores.

A segunda sprint (Banco de Dados), os grupos terão que desenvolver todo o banco de dados utilizado no projeto, sendo assim, terá que ter as tabelas, as colunas de cada tabela que se relacionam, enfim, efetuar uma modelagem de dados para mostrar aos professores.

A terceira sprint (Backend), os grupos terão que passar o Frontend para dentro do projeto Laravel, e aplicar o backend para que as coisas funcionem corretamente. (Lembrando que é uma obrigatoriedade que os projetos estejam em Laravel na data da entrega do projeto).

Como funcionam as Sprints ?
No caso do frontend, quando o módulo de HTML e CSS finaliza, é dado início a sprint de Frontend.
No caso do banco de dados, quando o módulo de banco de dados finalizar, comecaremos a sprint de banco de dados.
No caso do backend, funciona um pouco diferente, afinal, o tempo já está apertado devido estar no fim do curso e vocês terão que trabalhar adicionando os conhecimento adquiridos em Laravel no projeto, já durante o módulo.

As Sprints tem data limite para entrega. Ou seja, sempre que chegar a data limite os grupos deverão enviar o entregável para os professores analisarem e avaliarem, e como docentes, terão que dar um feedback para cada grupo.

Fases do Projeto Integrador*

- Front | Interface do Usuário
- Front | Interface do Administrador
- Front | Ajustes e adições
- Banco de Dados | Definição de Tabelas e Relacionamentos
- Back | Implantação do Laravel

*Resultado Final Esperado*

Um projeto onde seja possível:

- Se cadastrar, logar, deslogar e navegar pelo site;
- Acessar por desktop ou mobile;
- Acessar como administrador;
- Realizar as ações do CRUD - Criar, Ler, Atualizar e Deletar registros (produtos, usuários)
